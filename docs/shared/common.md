# 通用约定

> 本文档是「爱开发」全平台 API 的共享约定。各端接口文档默认遵循此规范，除非在具体接口中显式覆盖。

---

## 1. Base URL

| 环境 | 地址 |
|------|------|
| 开发 | `http://localhost:8080/api/v1` |
| 测试 | `https://api-test.lovedev.cn/api/v1` |
| 生产 | `https://api.lovedev.cn/api/v1` |

---

## 2. 认证

所有需登录的请求必须在 Header 中携带 Access Token：

```
Authorization: Bearer <access_token>
```

Token 获取方式见 [auth/](../auth/api.md)。

| 项目 | 值 |
|------|-----|
| Access Token 有效期 | 2 小时 |
| Refresh Token 有效期 | 30 天 |
| 刷新接口 | `POST /api/v1/auth/{role}/refresh-token` |

**401 处理**：前端拦截器在收到 `401 Unauthorized` 时应自动尝试刷新 Token；刷新失败则跳转登录页。

---

## 3. 统一响应格式

所有接口返回 JSON，结构如下：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `number` | 业务状态码，`0` 表示成功，非 `0` 表示异常 |
| `message` | `string` | 人类可读的提示信息（中文） |
| `data` | `any` | 实际载荷，可能为 `null` |

**列表接口**的 `data` 结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [ ... ],
    "total": 128,
    "page": 1,
    "pageSize": 20
  }
}
```

---

## 4. 分页

| 查询参数 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `page` | `number` | `1` | 页码，从 1 开始 |
| `pageSize` | `number` | `20` | 每页条数，最大 `100` |

---

## 5. HTTP 方法约定

| 方法 | 用途 |
|------|------|
| `GET` | 获取资源（无副作用） |
| `POST` | 创建资源或触发动作 |
| `PUT` | 全量更新资源 |
| `PATCH` | 部分更新资源（不常用，本项目统一用 `PUT` + 完整载荷） |
| `DELETE` | 删除资源 |

---

## 6. 时间格式

所有时间字段统一使用 **ISO 8601** 格式，时区为 `Asia/Shanghai` (UTC+8)：

```
2026-05-30T14:30:00+08:00
```

仅日期字段（无时间部分）使用 `YYYY-MM-DD`：

```
2026-05-30
```

---

## 7. 文件上传

- `Content-Type`: `multipart/form-data`
- 单文件大小上限：`10MB`（图片）、`50MB`（交付物压缩包）
- 文件上传接口独立于业务接口，返回文件 URL 后再将 URL 提交到业务接口

```http
POST /api/v1/upload
Content-Type: multipart/form-data

// 响应
{
  "code": 0,
  "data": {
    "url": "https://cdn.lovedev.cn/uploads/2026/05/abc123.png",
    "name": "abc123.png",
    "size": 204800,
    "mime": "image/png"
  }
}
```

## 8. 错误码

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `0` | `200` | 成功 |
| `400` | `400` | 请求参数错误 |
| `401` | `401` | 未认证 / Token 过期 |
| `403` | `403` | 无权限（角色不足 / 非本人操作） |
| `404` | `404` | 资源不存在 |
| `409` | `409` | 资源冲突（如重复抢单、重复实名认证） |
| `422` | `422` | 业务规则校验失败（如余额不足、等级不够） |
| `429` | `429` | 请求过于频繁 |
| `500` | `500` | 服务器内部错误 |
| `1001` | `422` | 余额不足 |
| `1002` | `422` | 开发者等级不够接该单 |
| `1003` | `422` | 项目状态不允许此操作 |
| `1004` | `422` | 今天已上传过进度 |
| `1005` | `409` | 该单已被其他开发者抢走 |
| `1006` | `403` | 消息包含敏感内容，已被 AI 审核拦截 |
| `1007` | `422` | 信誉分不足，无法使用分期支付 |
| `1008` | `422` | 未实名认证，请先认证 |
| `1009` | `429` | 验证码发送过于频繁，请 60 秒后重试 |
| `1010` | `422` | 提现金额超过可用余额 |
| `1011` | `422` | 未绑定提现账户 |
| `1012` | `409` | 该项目已存在进行中的争议 |
| `1013` | `403` | 未通过实名认证，无法进行此操作 |

---

## 9. 速率限制

| 端点类型 | 限制窗口 | 限制次数 | 说明 |
|----------|----------|----------|------|
| 发送验证码 | 60 秒 | 1 次 / 邮箱 | 同一邮箱同一场景 |
| 登录 | 60 秒 | 5 次 / IP | 防止暴力破解 |
| 注册 | 60 秒 | 3 次 / IP | 防止批量注册 |
| 抢单 | 1 秒 | 10 次 / 用户 | 防止脚本刷单 |
| AI 对话消息 | 1 秒 | 3 次 / 用户 | 防止滥用 AI 流 |
| 其他 REST 接口 | 1 秒 | 30 次 / 用户 | 通用频率限制 |

> 超限时返回 `429` + `"请求过于频繁，请稍后重试"`。

## 10. WebSocket 连接

即时通讯使用 WebSocket 协议，详见 [websocket/](../websocket/protocol.md)。

连接地址：

```
wss://ws.lovedev.cn/ws?token=<access_token>
```

## 10. 跨域（CORS）

| 来源 | 是否允许 |
|------|----------|
| `https://lovedev.cn` | ✓ |
| `https://admin.lovedev.cn` | ✓ |
| `https://dev.lovedev.cn` | ✓ |
| `https://*.lovedev.cn` | ✓ |
| `http://localhost:*` | ✓（仅开发环境） |

> 注意：全平台统一使用 `lovedev.cn`，不接受 `livedev.cn` 等变体。

允许的方法：`GET, POST, PUT, PATCH, DELETE, OPTIONS`

允许的请求头：`Authorization, Content-Type, X-Requested-With`
